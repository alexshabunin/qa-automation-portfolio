01 / Architectures

Three stacks. One application. Honest tradeoffs.

Track A

pytest + Playwright + POM

Classic Page Object Model with function-scope fixtures, per-test page.route() helpers, and dict literals for fake data. The stack 80% of teams already speak.

11 tests~12s
Track B

vedro + d42 + Webbricks

BDD-style steps, state-guaranteed contexts, typed MockedRoute with strict count checks, d42 schemas with source-cited bounds. Earns its weight at 1000+ tests.

7 scenarios~5s
API

requests + ApiManager

Facade over per-domain API clients, session/class/function fixture scopes, real JWT against a real FastAPI service. Every request becomes an Allure step with body attached.

25 tests~3s
02 / In motion

A test driving the SPA.

Open drawer · fill form · pick status + tags · save · card lands · toast. Same flow asserted across ui-pytest and ui-vedro.
03 / Real assertions

The badge counts every request. It says ×1.

The SPA's loadTasks is wrapped in a 300 ms debounce. Five keystrokes in 200 ms — the counter goes ×0 → ×1. Anything else fails the test with the exact URLs the mock saw.
